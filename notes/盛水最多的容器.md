# 盛最多水的容器

输入为一个长度为n的整数数组height，比如[1,8,6,2,5,4,8,3,7]
寻找0 <= i, j < len(height)，使得 abs(j - 1) * min(height[i], height[j])最大

此题主要的精彩之处在于*缩减搜索空间*的思想。
乍一看，i, j是需要o(n*n)的枚举的。
但如果i初始化在0，j初始化在len(height) - 1时。
我们会发现：
1、此时的情况有三种，要么a、height[left] < height[right]；要么b、height[left] > height[right]；要么c、height[left] = height[right]。
    且：在a情况下，矩形的宽度是最大的，此时若right -= 1，则新矩形的宽度减小，同时长度不会比现在大（木桶效应），所以此时left += 1是唯一选择。
    同理，在b情况下只有做right -= 1
    c情况下无论是left += 1或者是right -= 1面积都是减少的，所以二者选其一即可
